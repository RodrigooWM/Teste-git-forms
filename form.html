No form, atribuímos dois principais atributos que são "action" e "method". O primeiro diz respeito ao destino daquelas informações, já o segundo representa um verbo, podendo ser get ou post.

Get é usado para validação de dados em um form. Post é usado para criar novos dados de um usuário, por exemplo.

input do tipo "password" é usado para digitar senhas e devemos usar o maxlength e minlength para definir o número de caracteres da senha.

O atributo "size" define o tamanho do campo do input.

Pattern (padrão de senha complexa, por exemplo) é um atributo do input.

Podemos usar como exemplo o pattern [0-9a-fA-F]{4,8}. Ele quer dizer que a senha terá números entre 0 e 9, letras minúsculas e maiúsculas entre a e f. Sendo 4 números e 8 letras.

O atributo required torna o input essencial para avançar no formulário.

O inputmode="numeric" abrirá o teclado numérico no aparelho caso estejamos utilizando-o.

Podemos considerar o autocomplete="on", para que o navegador do usuário dê sugestões de preenchimento naquele campo. Alguns navegadores vem com o preenchimento automático, portanto se quisermos remover essa funcionalidade, basta deixarmos definido como "off".

O atributo "multiple" no input nos permite inserir mais de uma informação solicitada, separando-as por vírgula. Em se tratando de uma lista select, se usarmos o atributo multiple, poderemos marcar mais de uma alternativa ao mesmo tempo.

O "pattern", além de definir os caracteres também pode restringir uma terminação de e-amil, por exemplo, principalmente por se tratar de ferramentas corporativas.Ex.:
nome do funcionário@empresa.com → pattern=".+@empresa.com"
Dessa forma, apenas e-mails com essa terminação serão aceitos no campo.

O pattern para solicitar uma terminação .com.br é o seguinte:
pattern=".*\.com\.br\/.*"

Outro exemplo:
pattern="(?=.*\d)(?=.*[a-z])(?=.*[A-Z]).{8,}"
O exemplo acima diz respeito a um input com 8 ou mais caracteres, com pelo menos um número, uma letra maiúscula e uma minúscula.

o atributo do "spellcheck", que pode ser true ou false, dentro do input faz a verificação ortográfica.

Fique atento pois alguns navegadores vem com a verificação ortográfica pré-definida, portatno, talvez só tenhamos que desabilitar.

Input files; atributos:

  • Value - contém o arquivo a ser enviado
  • Accept - descreve que tipos de arquivos serão aceitos (.doc, .docx, .pdf, etc)
  • Files - Lista de arquivos
  • Multiple - aceita vários arquivos de uma só vez. Também pode ser usado em outros tipos de input, além de file.

O input tipo "color" serve para selecionarmos uma cor. Podemos adicionar opções de cores usando um datalist também, pois a maioria dos navegadores ainda dá suporte a esse tipo de tag.

Checkbox vs Radio

A marcação do checkbox é recebida no back-end como "subscribe on", do contrário, não tem nenhum retorno, já que ela não foi marcada. Já o input radio, seguirá a ordem, 0,1,2,3... para representar qual opção foi marcada.

Entendendo o Textarea

TextArea → Se deixarmos a tag em linha, o cursor, ao clicar no elemento, ficará no início do campo de texto. Se deixarmos em coluna, no código, o cursor ficará onde clicarmos dentro da caixa (no meio, fim, começo, etc).

O atributo "rows" e "cols" definirá o número de linhas e colunas que o campo terá inicialmente (lembrando que seu tamanho ainda poderá ser alterado na página).

Por padrão do navegador, normalmente o textarea tem o atributo "wrap=on", para que o texto seja quebrado e passe para a linha seguinte. Caso queiramos desabilitar, basta definir como off. Não esqueça de usar por via das dúvidas.

Select

Recomendamos inserir como primeiro option, uma espécie de placeholder. Assim, termos uma orientação do que devemos escolher naquele campo.

Optgroup

Essa tag cria um agrupamento de opções dentro de um elemento.

Essa tag basicamente é uma sublista dentro do select. O select representa uma lista, o optgroup organiza a lista pos categorias. A legenda se dá por meio do label

Construção do código:
< select >
<optgrpup label="primeira lista">
< option value=""> Opção 1.1 </option >
</optgrpup>
<optgrpup label="segunda lista">
< option value=""> Opção 1.2 </option >
</optgrpup>
<optgrpup label="terceira lista">
< option value=""> Opção 1.3 </option >
</optgrpup>
< / select >

Input search

Area-label → Esse atributo é adicionado dentro do input e tem a mesma finalidade do "label for", de acessibilidade. Mas, definimos o que será lido pela ferramenta de acessibilidade através do comando: area-label="o que será lido"

É recomendado usar o pattern para restringir a busca em se tratando de campo de dados, de informações sensíveis, ou de informações específicas.

Curiosidade: Dentro do input, podemos definir um pattern e um title em sequencia. Dessa forma, ao exibir a mensagem de validação negativa, as duas mensagem serão apresentadas juntas, a automática e a que definimos no title.

Number, Range, Data e hora

Input do tipo range, é uma telinha para aumentar e diminuir. Podemos definir maximo, minimo, step, valor inicial (value) e orientação. Essa última só funcionará em alguns navegadores.

Input date, é possível inserir um value, lembrando que o formato é AAAA-MM-DD

Input datetime-local nos permite inserir data completa e hora.
